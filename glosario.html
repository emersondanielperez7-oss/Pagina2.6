01
IdempotencyAPI / Distributed Systems
IdempotenciaTraducción
 EN: A design property of HTTP methods and distributed systems where an identical request can be executed multiple times sequentially without changing the system state beyond the initial application. Crucial for fault-tolerant architectures and retry mechanisms (e.g., payment gateways) to prevent duplicate operations.

ES: Propiedad de diseño de los métodos HTTP y sistemas distribuidos donde una solicitud idéntica puede ejecutarse múltiples veces de forma secuencial sin alterar el estado del sistema más allá de la aplicación inicial. Crucial para arquitecturas tolerantes a fallos y mecanismos de reintento (ej. pasarelas de pago) para evitar operaciones duplicadas.
ID Safe Retries
02
Immutable InfrastructureDevOps / Cloud
Infraestructura InmutableTraducción
 EN: An operations management paradigm where infrastructure components (such as servers or containers) are replaced entirely rather than modified or upgraded in place after deployment. This eliminates configuration drift, guarantees environment consistency across lifecycles, and simplifies rollback vectors.

ES: Paradigma de gestión de operaciones donde los componentes de infraestructura (como servidores o contenedores) se reemplazan por completo en lugar de modificarse o actualizarse en el sitio después de su despliegue. Esto elimina la desviación de configuración (configuration drift), garantiza la consistencia del entorno y simplifica los vectores de reversión (rollbacks).
IM Replace, Don't Patch
03
IndexDatabase Engines
ÍndiceTraducción
 EN: An auxiliary, persistent data structure (typically a B-Tree or Hash map) that optimizes query performance by minimizing the number of disk block access overhead operations required to locate specific target rows. While accelerating read constraints, indices introduce a computational trade-off by adding overhead to write operations.

ES: Estructura de datos auxiliar y persistente (típicamente un árbol B-Tree o mapa Hash) que optimiza el rendimiento de las consultas al minimizar el número de operaciones de acceso a bloques de disco requeridas para localizar filas específicas. Aunque acelera las lecturas, introduce una penalización computacional en las operaciones de escritura.
IX Fast Data Lookup
04
Inversion of Control (IoC)Design Patterns
Inversión de ControlTraducción
 EN: An architectural design principle that externalizes the program execution flow control, delegating object lifecycle management and dependency orchestration to a specialized framework engine. Frequently implemented via Dependency Injection (DI) patterns to enforce loose coupling constraints.

ES: Principio de diseño de software que externaliza el control del flujo de ejecución del programa, delegando la gestión del ciclo de vida de los objetos y la orquestación de dependencias a un framework especializado. Frecuentemente implementado mediante patrones de Inyección de Dependencias (DI) para imponer un acoplamiento débil.
IC Loose Coupling
05
IP Address (Internet Protocol Address)Networking
Dirección IPTraducción
 EN: A unique numerical identifier assigned to each node interface participating within a computer network utilizing the Internet Protocol suite. It structures the global routing topography and logical endpoint targeting, distributed predominantly as 32-bit (IPv4) or 128-bit (IPv6) architectures.

ES: Identificador numérico único asignado a cada interfaz de nodo participante en una red informática que utiliza la suite de protocolos de Internet. Estructura la topografía de enrutamiento global y el direccionamiento lógico de puntos finales, distribuida predominantemente en arquitecturas de 32 bits (IPv4) o 128 bits (IPv6).
IP Network Addressing
