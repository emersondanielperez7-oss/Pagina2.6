01
Microservices ArchitectureSoftware Architecture
Arquitectura de MicroserviciosTraducción
 EN: An architectural style that structures an application as a collection of loosely coupled, autonomously deployable, and small services organized around business capabilities, each running its own process and managing an isolated database.

ES: Estilo arquitectónico que estructura una aplicación como una colección de servicios pequeños, acoplados de forma débil y desplegables de manera autónoma. Cada servicio se organiza en torno a una capacidad de negocio, ejecuta su propio proceso y gestiona una base de datos aislada.
MS Independent Services
02
MiddlewareDistributed Systems
Middleware / Software de MediaciónTraducción
 EN: A functional software layer positioned between the operating system and distributed applications. It acts as "connective tissue" to enable cross-platform communication, data translation, and asynchronous request routing without exposing underlying infrastructure.

ES: Capa de software funcional posicionada entre el sistema operativo y las aplicaciones distribuidas. Actúa como "tejido conectivo" para permitir la comunicación multiplataforma, la traducción de datos y el enrutamiento asíncrono sin exponer la infraestructura subyacente.
MW Connective Tissue
03
Monolithic ArchitectureSoftware Architecture
Arquitectura MonolíticaTraducción
 EN: A traditional software development model where the entire application—including user interface, business logic, and data access components—is composed into a single, indivisible codebase and deployed as a unified runtime unit.

ES: Modelo tradicional de desarrollo de software donde toda la aplicación (incluyendo interfaz de usuario, lógica de negocio y componentes de acceso a datos) se compone en una única base de código indivisible y se despliega como una unidad de ejecución unificada.
MO Unified Codebase
04
MVCC (Multi-Version Concurrency Control)Database Engines
Control de Concurrencia Multi-VersiónTraducción
 EN: A database optimization method that allows concurrent reads and writes without mutual locking. It achieves high performance by maintaining multiple physical versions of a data record simultaneously, trading storage space for lock-free read transactions.

ES: Método de optimización de bases de datos que permite lecturas y escrituras concurrentes sin bloqueos mutuos. Logra un alto rendimiento manteniendo múltiples versiones físicas de un registro simultáneamente, sacrificando espacio de almacenamiento a cambio de transacciones de lectura libres de bloqueos.
MV Lock-free Reads
05
Materialized ViewDatabase Management
Vista MaterializadaTraducción
 EN: A database object that physically stores the precomputed results of a query on disk. Unlike standard virtual views, it dramatically accelerates heavy analytical reads but requires a strict refresh strategy (sync or async) to handle data drift.

ES: Objeto de base de datos que almacena físicamente en disco los resultados precalculados de una consulta. A diferencia de las vistas virtuales estándar, acelera drásticamente las lecturas analíticas pesadas pero requiere una estrategia estricta de refresco (síncrona o asíncrona) para manejar el desajuste de datos.
VW Precomputed Disk Cache
06
Multi-Tenant ArchitectureCloud Architecture
Arquitectura Multi-InquilinoTraducción
 EN: A software architecture paradigm where a single, shared instance of a software application serves multiple distinct user groups or customers (tenants). It strictly isolates client data at the logic or database level while sharing compute infrastructure.

ES: Paradigma arquitectónico donde una única instancia compartida de una aplicación sirve a múltiples clientes o grupos de usuarios distintos (inquilinos). Aísla estrictamente los datos de cada cliente a nivel lógico o de base de datos mientras comparten la infraestructura de cómputo.
MT Shared Instance
07
Message QueueIntegration Patterns
Cola de MensajesTraducción
 EN: An asynchronous communication buffer used in distributed systems to store messages sequentially until processed by a consumer. It decouples data producers from consumers, ensuring reliable data transfer even during traffic spikes or receiver downtime.

ES: Búfer de comunicación asíncrona utilizado en sistemas distribuidos para almacenar mensajes secuencialmente hasta ser procesados por un consumidor. Desacopla los productores de datos de los consumidores, garantizando la transferencia de datos incluso durante picos de tráfico.
MQ Asynchronous Buffer

NormalizaciónTraducción
 EN: An algorithmic and mathematical process in relational database design that structures tables according to Normal Forms (1NF through BCNF/5NF) to systematically eliminate data redundancy and prevent modification anomalies (insertion, update, and deletion).

ES: Proceso de optimización algorítmica y matemática en el diseño de bases de datos relacionales que estructura las tablas según Formas Normales (1FN a BCNF/5FN) para eliminar sistemáticamente la redundancia de datos y prevenir anomalías operativas (inserción, actualización y borrado).
NRM Anomaly Prevention
02
NoSQL (Not Only SQL)Database Engines
NoSQLTraducción
 EN: A broad classification of non-relational database management systems designed to bypass the structural rigidities of SQL paradigms. They optimize horizontal scalability, high availability, and schema flexibility across distributed clusters by utilizing specialized data models (Document, Key-Value, Column-family, Graph).

ES: Clasificación amplia de sistemas de gestión de bases de datos no relacionales diseñados para evadir las rigideces estructurales del paradigma SQL. Optimizan la escalabilidad horizontal, la alta disponibilidad y la flexibilidad de esquemas en clústeres distribuidos utilizando modelos de datos especializados (Documental, Clave-Valor, Familias de Columnas, Grafos).
NQ Horizontal Scaling
03
NodeDistributed Systems
NodoTraducción
 EN: A discrete, independent computational or storage unit (such as a physical server, virtual machine, or container) within a distributed cluster. It executes localized software execution, processes distinct workloads, and communicates with peers via network consensus protocols.

ES: Unidad de almacenamiento o cómputo discreta e independiente (como un servidor físico, máquina virtual o contenedor) dentro de un clúster distribuido. Ejecuta software localizado, procesa cargas de trabajo específicas y se comunica con sus pares mediante protocolos de consenso de red.
ND Cluster Unit
04
Network PartitionFault Tolerance
Partición de RedTraducción
 EN: An architectural failure condition in a distributed system where communication links between nodes are broken, splitting the cluster into isolated sub-networks. This state forces the architecture to choose between consistency or availability according to the CAP theorem.

ES: Condición anómala de fallo en un sistema distribuido donde los enlaces de comunicación entre nodos se rompen, dividiendo al clúster en subredes aisladas. Este estado fuerza a la arquitectura a elegir entre consistencia o disponibilidad bajo los términos del teorema CAP.
NP CAP Split-Brain
05
N-Tier ArchitectureArchitectural Patterns
Arquitectura N-CapasTraducción
 EN: A client-server architectural pattern where software components are segregated into 'N' distinct logical or physical layers (typically presentation, application logic, and data storage). This enforces a strict separation of concerns at the cost of additional network call latency.

ES: Patrón arquitectónico cliente-servidor donde los componentes de software se segregan en 'N' capas lógicas o físicas distintas (típicamente presentación, lógica de aplicación y almacenamiento de datos). Impone una estricta separación de conceptos a costa de añadir latencia por llamadas de red.
NT Layered Isolation

Mapeo Objeto-RelacionalTraducción
 EN: An abstraction layer that translates data between incompatible type systems: object-oriented programming languages and relational databases. It mitigates the object-relational impedance mismatch but can introduce performance overhead via unoptimized SQL generation, such as the N+1 query problem.

ES: Capa de abstracción que traduce datos entre sistemas de tipos incompatibles: lenguajes orientados a objetos y bases de datos relacionales. Mitiga el desajuste de impedancia objeto-relacional, pero puede introducir sobrecargas de rendimiento debido a la generación de SQL no optimizado, como el problema de consulta N+1.
OM Impedance Bridge
02
OLTP (Online Transaction Processing)Database Paradigm
Procesamiento de Transacciones en LíneaTraducción
 EN: A data processing paradigm optimized for executing massive volumes of rapid, highly concurrent transaction workloads (typically CRUD operations). It strictly enforces ACID properties to maintain operational data integrity in real-time environments like banking systems or checkout funnels.

ES: Paradigma de procesamiento de datos optimizado para la ejecución de volúmenes masivos de cargas de trabajo transaccionales rápidas y altamente concurrentes (típicamente operaciones CRUD). Impone estrictamente las propiedades ACID para mantener la integridad operativa en entornos en tiempo real como sistemas bancarios o pasarelas de pago.
TP High-Concur. ACID
03
OLAP (Online Analytical Processing)Data Warehousing
Procesamiento Analítico en LíneaTraducción
 EN: A data architecture optimized for high-performance evaluation of complex, multidimensional analytical queries over large-scale historical datasets. It utilizes star or snowflake schemas and precomputed cubes to facilitate fast aggregations, sacrificing real-time write speeds for analytical throughput.

ES: Arquitectura de datos optimizada para la evaluación de alto rendimiento de consultas analíticas complejas y multidimensionales sobre conjuntos de datos históricos masivos. Utiliza esquemas en estrella o copo de nieve y cubos precalculados para facilitar agregaciones rápidas, sacrificando velocidad de escritura por rendimiento analítico.
AP Multi-Dim Cubes
04
Optimistic Concurrency Control (OCC)Database Engines
Control de Concurrencia OptimistaTraducción
 EN: A lock-free concurrency management method that assumes resource conflicts are rare. Transactions execute reads and writes freely in isolation, and only verify structural state violations or version drifts during the final commit phase, triggering a rollback if a collision is detected.

ES: Método de gestión de concurrencia libre de bloqueos que asume que los conflictos son raros. Las transacciones ejecutan lecturas y escrituras libremente de forma aislada, y solo verifican violaciones de estado o desvíos de versión durante la fase final de confirmación (commit), ejecutando un rollback si se detecta colisión.
OC Commit Validation
05
Object StorageCloud Infrastructure
Almacenamiento de ObjetosTraducción
 EN: A data storage architecture designed for unstructured data that manages information as discrete objects within a flat namespace rather than a hierarchical file tree. Each object wraps raw data, robust customizable metadata, and a unique identifier to provide massive scale-out capabilities.

ES: Arquitectura de almacenamiento diseñada para datos no estructurados que gestiona la información como objetos discretos dentro de un espacio de nombres plano, en lugar de un árbol de archivos jerárquico. Cada objeto envuelve datos puros, metadatos robustos personalizables y un identificador único para una escalabilidad horizontal masiva.
OS Flat Namespace
